## ğŸ”„ Agent Pipeline

### Pipeline Execution Flow

1. **Requirement Analysis** (Step 1)
   - Parses natural language input
   - Generates structured specifications
   - Outputs: `requirements.md`

2. **Code Generation & Review Loop** (Steps 2-3)
   - Generates Python code
   - Reviews code for quality and security
   - Iterates up to 3 times if revisions needed
   - Outputs: `generated_code.py`, `code_review.md`

3. **Documentation Generation** (Step 4)
   - Creates comprehensive documentation
   - Includes API reference and usage examples
   - Outputs: `documentation.md`

4. **Test Case Generation** (Step 5)
   - Generates pytest test suite
   - Includes happy path, edge cases, and error handling tests
   - Outputs: `test_generated_code.py`

5. **Deployment Configuration** (Step 6)
   - Creates deployment files
   - Outputs: Dockerfile, docker-compose.yml, requirements.txt

6. **UI Generation** (Step 7)
   - Generates Streamlit interface (optional)
   - Outputs: `streamlit_ui.py`

### Feedback Loop Mechanism

```python
max_iterations = 3
iteration = 0
code_approved = False

while not code_approved and iteration < max_iterations:
    iteration += 1
    code = coding_agent.generate_code(requirements, feedback)
    review = review_agent.review_code(code, requirements)
    
    if review_agent.is_approved(review):
        code_approved = True
    else:
        feedback = review  # Send feedback for next iteration
```

## ğŸ“š API Reference

### AgentPipeline

Main orchestration class for the multi-agent framework.

#### Constructor

```python
AgentPipeline(api_key: str = None, model: str = None)
```

**Parameters:**
- `api_key` (str, optional): OpenAI API key. Defaults to environment variable.
- `model` (str, optional): Model name. Defaults to config value.

#### Methods

##### execute_pipeline

```python
execute_pipeline(
    user_requirement: str,
    save_outputs: bool = True
) -> Dict[str, Any]
```

Execute the complete multi-agent pipeline.

**Parameters:**
- `user_requirement` (str): Natural language requirement
- `save_outputs` (bool): Whether to save outputs to files

**Returns:**
- `Dict[str, Any]`: Dictionary containing:
  - `user_requirement`: Original input
  - `timestamp`: Execution timestamp
  - `outputs`: Dict with all agent outputs
  - `metadata`: Execution metadata (iterations, status, etc.)

**Example:**
```python
pipeline = create_pipeline(api_key="sk-...")
results = pipeline.execute_pipeline(
    "Create a REST API for managing books"
)

print(f"Status: {results['metadata']['status']}")
print(f"Code: {results['outputs']['code']}")
```

### Individual Agents

All agents follow a similar interface:

```python
# Requirement Agent

# Coding Agent
```

**Solution**:

3. Ensure environment variables are loaded

#### Import Errors

**Problem**: `ModuleNotFoundError: No module named 'src'`

**Solution**:
```bash
# Ensure you're in the project root
export PYTHONPATH="${PYTHONPATH}:$(pwd)"

# Or add to .env
echo "PYTHONPATH=." >> config/.env
```

#### Streamlit Not Starting

**Problem**: Streamlit fails to start

**Solution**:
```bash
# Check if port 8501 is available
netstat -an | grep 8501

# Use different port
streamlit run ui/streamlit_app.py --server.port=8502
```

#### Output Files Not Saving

**Problem**: Generated files not appearing in output directory

**Solution**:

```bash
# Check directory permissions

```
